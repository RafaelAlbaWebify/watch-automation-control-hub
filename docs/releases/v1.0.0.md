# WATCH v1.0.0

WATCH v1.0.0 is the first stable release candidate of the Workflow Automation & Technical Control Hub.

## Included capabilities

- Local target inventory and timezone-aware interval schedules.
- Deterministic due-occurrence planning and atomic claims.
- Bounded one-shot execution with at-most-once markers.
- HTTP, DNS, redirect, timing, title, header, body-size, and TLS-expiry evidence.
- Previous-run comparison, deterministic findings, actions, and reports.
- Missed and stale occurrence attention visibility.
- Explicit operator-controlled retry attempts with immutable original evidence.
- FastAPI operator API and read-only browser workbench.
- Windows PowerShell setup, verification, execution, export, and Task Scheduler lifecycle.

## Safety boundary

- Explicit targets only.
- HTTP and HTTPS only.
- Public-address validation and address-pinned transport.
- Bounded timeouts, redirects, work selection, lookback, grace periods, and retries.
- No crawling, authentication bypass, form submission, stored credentials, SYSTEM task, automatic retry, stale-work recovery, or external-system modification.

## Verification

The release candidate is proven by:

- Ruff and strict mypy.
- Full pytest suite on Linux and Windows.
- Clean-checkout Windows setup and operator workflow.
- Windows review ZIP export.
- Chromium semantic and screenshot regression.
- Playwright desktop/mobile human-style review.
- Temporary real Windows Task Scheduler install, invocation, verification, uninstall, and cleanup.
- One approved bounded live collection against `https://example.com` with a sanitized committed report.
- Automated V1 release-gate evidence.

## Known limitations

- Local-first file persistence; no multi-user database or authentication.
- One bounded foreground process per scheduled trigger.
- No automatic recovery of interrupted executing occurrences.
- No external notifications or third-party system writes.
- Findings are deterministic operational indicators, not confirmed root causes.
