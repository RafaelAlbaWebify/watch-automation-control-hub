# WATCH V1 release checklist

The former manual validation activities are now automated wherever hosted infrastructure can reproduce the real behavior honestly. The evidence is generated by the `WATCH V1 Release Validation` workflow.

## 1. Windows scheduler lifecycle

- [x] Generate and review the task manifest.
- [x] Install a uniquely named WATCH task on a real hosted Windows runner.
- [x] Verify current-user limited privilege, no stored credentials, and `IgnoreNew` overlap prevention.
- [x] Start the installed task and retain its scheduled-run JSON evidence.
- [x] Uninstall the temporary task in an unconditional cleanup step.
- [x] Confirm the task no longer exists after uninstall.

This validates the full Windows Task Scheduler lifecycle in a disposable Windows environment. Deployment to a particular long-lived workstation is optional and follows the same verified commands.

## 2. Approved live-run example

- [x] Use the explicitly approved low-risk target `https://example.com`.
- [x] Run one bounded read-only collection.
- [x] Sanitize dynamic run identifiers, timestamps, action identifiers, and resolved addresses.
- [x] Verify that the public report contains the approved target and no raw resolved address.
- [x] Commit the representative report at `docs/examples/example-com-live-report.md`.

## 3. Clean-checkout and first-run review

- [x] Start from a clean hosted Windows checkout with no `.venv` or `.watch-data`.
- [x] Run the documented setup, verification, and demo commands.
- [x] Exercise planning, one-shot execution, and scheduler planning without a live network request.
- [x] Launch the workbench against deterministic evidence.
- [x] Review all operator pages through Playwright at desktop and mobile viewports.
- [x] Confirm keyboard skip-link behavior and zero browser console errors.

## 4. Portfolio presentation review

- [x] Verify the README positioning and safety claims used for portfolio presentation.
- [x] Capture every operator page at desktop and mobile sizes.
- [x] Select the final portfolio screenshot shortlist:
  - dashboard;
  - target inventory;
  - change timeline;
  - operational actions.
- [x] Produce machine-readable portfolio-review evidence.

## 5. Stable release decision

- [x] Confirm automated repository readiness is `PASS`.
- [x] Confirm the human-like release-validation gate is `PASS`.
- [x] Confirm the approved live evidence is sanitized and committed.
- [x] Confirm temporary scheduler cleanup is successful.
- [x] Set the stable version to `0.1.0` and publish the `v0.1.0` tag and GitHub Release.
- [x] Verify the published release is not a draft or prerelease.
- [x] Verify the non-empty `WATCH-v0.1.0-windows.zip` release asset.

The WATCH v0.1.0 release checklist is complete and backed by repository-native machine-readable evidence.
